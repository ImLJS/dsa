## Difficulty Progression

For each new pattern, follow this order:
1. Read the pattern page in the PDF — understand the template
2. Solve 2–3 Easy problems to build muscle memory
3. Solve 3–4 Medium problems without hints
4. Attempt 1 Hard problem — look at hints after 30 min
5. Re-solve any problem that took more than 25 min without looking at the solution

---

## Red Flags During Practice

If you notice these, slow down and revisit the fundamentals:

- Reaching for brute force every time → pattern recognition is weak, drill the cheatsheet
- Solving Mediums fine but Hards feel impossible → practice combining 2 patterns
- Getting the approach right but wrong code → slow down, trace through with a small example
- Timing out → check your data structures (`list.pop(0)` instead of `deque.popleft()`)
- Correct logic but off-by-one errors → always test with arrays of length 0, 1, and 2
